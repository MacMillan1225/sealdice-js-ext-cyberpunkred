## 一、核心概念：电梯比喻

网络空间被比喻为**电梯系统**：
- 每一层 = 一个楼层
- 每层有一扇"门"，门后可能是：程序、黑冰、另一名网行者、文件、控制节点等
- **关键规则**：
  - ✅ 每回合可**向下穿越任意数量楼层**（前提未被密码/障碍阻拦）
  - ❌ **不能跳过任何楼层**（必须逐层通过）
  - 必须按顺序打开每一扇门
  - 许多门后藏有**黑冰**——开门够快能躲开，否则触发攻击并进入先攻队列顶端

## 三、主持人分步构建指南

### 第0步：难度等级

| 参数 | 基础难度 | 标准难度 | 罕见难度 | 高级难度 |
|:---|:---:|:---:|:---:|:---:|
| 密码/文件/控制节点 DV | DV6 | DV8 | DV10 | DV12 |
| 有一线胜算的网行者交互界面 | 2 | 4 | 6 | 8 |
| 随时可能毙命的网行者的交互界面 | N/A | 2或以下 | 4或以下 | 6或以下 |

### 第1步：构筑网络空间

| 步骤 | 规则 |
|:---|:---|
| 总楼层数 | 掷 **3d6** |
| 分支判定 | 掷 **d10**，结果为 **7+** 则存在一个分支 |
| 额外分支 | 再次掷 d10，继续判定是否出现新分支，无则停止 |
| 分支位置 | **主干第二层之后**才可分支 |
| 分配原则 | 将总层数分配给主分支和所有分支 |
| 底部要求 | 确保存在一条**最长分支**作为明确"底部"，网行者可在此留病毒 |

### 第2步：填充网络空间

| 区域 | 掷骰规则 |
|:---|:---|
| **前两层（大厅）** | 在"大厅"表格上掷骰确定内容，若掷出已有程序/密码则**重新投掷** |
| **其余楼层** | 根据设定的难度等级，在对应"网络空间结构"表格上掷骰，若掷出已有程序/密码则**重新投掷** |

> 越深越有价值：网行者钻得越深，越有可能找到有价值的文件或有用的控制节点。

### 第3步：让空间融入世界

- 每个文件/控制节点必须在**肉身空间**中有对应存在的理由
- 文件和控制节点是"宝藏"，需对游戏进程和现实世界产生切实意义
- **控制节点不必以战斗为导向**，示例：
  - 摄像头、喷淋头、警报器、流水线传送带、视频显示终端、氯化控制阀、电子门锁、按摩椅、自动贩卖机、空调、自动投球机

**恶魔驻守规则**：
- 存在控制节点 → 需决定是否有**"恶魔"**操纵
- 任何主动防御系统都需要恶魔驻守才能运作
- 恶魔应与其位置匹配（小鬼最常见，本地电影院不会有"巴尔隆"除非藏有猫腻）

**自定义规则**：
- 可将生成的任何程序替换为**同等成本**的程序
- 可随意调换楼层位置
- 可跳过随机投骰，直接挑选想要的楼层（定制架构，不破坏平衡）
- 可从比当前难度**高一档**的空间中挑选楼层，打造**"Boss楼层"**

## 六、恶魔系统

### 恶魔基础规则

| 特性 | 说明 |
|:---|:---|
| 本质 | 非战前"真人工智能"，而是**特定的黑冰智能系统**（第四次企业战争后受严格限制） |
| 设计目的 | 专门攻击**肉身空间**目标（如网行者支援小队），利用无人机、炮塔等控制节点 |
| 属性 | 拥有**交互界面**和**战斗数值**（属性+技能） |
| 攻击方式 | 掷出的1d10加到战斗数值上 |
| 限制 | **无法运行程序或黑冰** |
| 可用能力 | 仅2种网行者能力：**电击**（自我防御）和**控制**（操作控制节点） |
| 控制节点 | 即使由恶魔操控，每个控制节点每回合**只能激活一次**；可控制**多个**控制节点 |
| 感知特性 | 对空间每个角落了如指掌，**不受密码限制**，始终能察觉网行者存在，速度对抗中**自动获胜** |
| 速度/感知 | **无SPD/PER数值**——无法从恶魔身边溜走，被发现时不会获得免费攻击机会 |
| 防御 | **无DEF**，像网行者一样使用**交互界面 + 1d10**进行防御 |
| 先攻规则 | 通过观测摄像头发现入侵者，或有网行者进入其网络空间时，**直接插入先攻队列最前端** |
| 行动优先级 | 优先使用网络动作**触发控制节点**，仅在**有剩余网络动作时**才用电击攻击敌方网行者 |
| AI特性 | 拥有特定且非自主的"智能"，类似电子游戏AI，拥有**"触发器"**设定预定行动 |

### （一）主动防御

**基础规则**：
- 能在区域内**独立行动**的小型机器人（如自动驾驶载具）
- 远程追踪目标，通常在防御地点周边范围（大楼、封闭区域）
- **除非由网行者手动控制，否则需要恶魔在空间中控制其节点才能正常运作**
- 未被控制时：易受"电子/安防技术"技能反制；反制后，在网络空间任一访问点6米/码范围内，可通过智能代理消耗一个动作远程控制
- 被网行者控制时：所有攻击和防御使用网行者自身技能

### （二）固定防御设施

**基础规则**：
- **无法移动**，仅限单一空间/区域内运作
- 被视为"主动"防御设施（程序需满足预设条件才触发攻击）
- 每个设施连接网络空间中的**独立控制节点**
- **不需要恶魔或网行者操作**
- 自行运作时：使用整合属性与技能的战斗数值 + 1d10
- **无法闪避攻击**（静止不动）
- 未被操控时：极易受电子/安防技术反制；反制后在网络空间访问点6米/码范围内可远程控制
- 被网行者控制时：所有攻击使用网行者自身技能

| 类型 | 描述 | 默认触发条件 | 说明 |
|:---|:---|:---|:---|
| **自动血雾群** | 自动武器将纳米机器人以红雾喷进房间；吸入后凝结血红蛋白从内部攻击；能过滤毒气攻击的装备可阻挡；防御区域内所有血肉之躯必须通过DV15"抵抗拷问/药物"检定，失败受3d6生命值伤害（护甲不损耗）；DV21电子/安防技术，需5分钟反制 | 目标在未佩戴有效通行证或工牌的情况下进入房间 | 对所有目标进行单次攻击 · 防卫区域边界 |

### （三）环境防御系统

**基础规则**：
- "内嵌"在特定日常环境中的系统
- 分布在有限空间（走廊、房间、较小封闭/围栏区域）
- 多为地板、墙壁、天花板结构的一部分，或内置发射器/投影仪
- 每个系统在网络空间中有**各自的控制节点**
- **全部自动**，即便由恶魔或网行者控制，**只能切换开或关**

|:---|:---|
| 无网络空间的电子设备 | 模拟信号设备（如烤面包机）或完全依赖城市网和记忆芯片运行的设备（智能代理终端、摄像头） |
| 应对方式 | 可用**电子/安防技术**技能应对，但**无法通过界面身份能力入侵** |
| 义体 | 同样无法通过界面能力入侵，由**义体技术技能**管辖 |
| GM注意事项 | 留意架设网络空间的运营方类型；简单空间（如夜店音响灯光）可能只有几个控制节点+小鬼程序，安全防护不在优先项 |
| 现实逻辑 | 赛博朋克世界中的人也在预算和运营需求之间找平衡来搭建系统 |
